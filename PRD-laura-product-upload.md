## 2. קיבוץ למוצר + מידות (Variants)

**המבנה:** צבע = מוצר נפרד. מידה = וריאנט תחת אופציה **"מידה"**. כל מידה = שורה/מק"ט נפרד בספק.
**המידה יכולה להיות בכל מקום ב-`תיאור פריט`** — לא רק בסוף (אומת בוולידציה, §9). דוגמאות:

| תיאור פריט בספק | → title | → מידה |
|---|---|---|
| מכנסיים סרוגים חום **XS** | מכנסיים סרוגים חום | XS *(מידה בסוף)* |
| אוברול קצר עם קשירה **NB** טרי ניוד | אוברול קצר עם קשירה טרי ניוד | NB *(מידה באמצע!)* |
| ציפית לכרית **34*44 ס"מ** אופוואיט COZY | ציפית לכרית אופוואיט COZY | 34*44 *(מטרי + "ס\"מ")* |

**אלגוריתם (מתוקן):**
1. סרוק את טוקני `תיאור פריט`; אם אחד מהם תואם לאוצר-המידות (§2.1) → אותו טוקן = **המידה**; הסר אותו (וגם "ס\"מ" צמוד) → מה שנשאר, אחרי כיווץ רווחים = **הכותרת**.
2. שורות עם אותה כותרת (אחרי החיתוך) = **מוצר אחד**, עם וריאנט לכל מידה.
3. אם אין טוקן-מידה → **מוצר וריאנט-יחיד** (רוב המוצרים — 513/606), הכותרת = כל התיאור.
4. **מידה מטרית (`34*44`, `75/100`) — כלל קיבוץ:** מידה מטרית היא וריאנט **רק אם אותו בסיס (שם ללא המידה) מופיע עם ≥2 מידות שונות**; אם היא יחידה — נשארת בשם (מוצר וריאנט-יחיד). מידת **ביגוד** (NB/XS/0-3…) — **תמיד** וריאנט גם אם יחידה.
   - הבחנה ביגוד↔מטרי: טווח מספרי ששני צדדיו ≤24 = ביגוד (חודשים); אחרת מטרי. "ס\"מ" צמוד — מסירים יחד עם המידה.
   - ⚠️ "ס\"מ" **אינו** סימן אמין (מפריך את עצמו) — הקיבוץ הוא הסימן. דיוק הכלל: **99.6%** (§9). ~0.4% הנותרים = אי-עקביות בנתוני הספק/האתר, לבדיקה ידנית.

### 2.1 אוצר המידות (מתוך הנתונים הקיימים)
- ביגוד: `NB · XS · 0-3 · 3-6 · 6-9 · 9-12 · 6-9M · 9-12M · 12-18M`
- מטרי (מצעים/כריות): `34*44 · 70*50 · 75/100 · 80/120`
- ⚠️ טעויות הקלדה בספק לתיקון/טיפול: `6-3 · 3-0`

## 3. קטגוריה מול תת-קטגוריה (שיוך לאוספים)

| רמה | איפה | ערך |
|---|---|---|
| **קטגוריה** (על) | Collection — `POST /collects.json` | **תמיד** `טקסטיל וביגוד לתינוק` (id **`477920559358`**) — 69/69 המשפחות ✓ |
| **תת-קטגוריה** | `product_type` + `tags` | `תאור משפחה` |
| **תת-קטגוריה (אוסף)** | Collection נוסף | לפי **טבלת המיפוי בנספח A** (משפחה → אוסף) |

> אין Smart Collections בחנות — האוטומציה **חייבת** ליצור `collects` במפורש לשני האוספים.
>
> **פתרון מזהי אוספים (חסר בגרסה קודמת):** נספח A נותן **שמות** אוספים, אך `collects.json` דורש `collection_id`. לכן האוטומציה: `GET /custom_collections.json` → בונה מיפוי שם→id; אם אוסף תת-הקטגוריה **לא קיים** → `POST /custom_collections.json {title}` ליצירתו → מסמן `is_new_collection` על המוצר לבדיקה. קטגוריית העל (`477920559358`) ידועה מראש.

## 4. תבנית התיאור (`body_html`)
הטקסט הגולמי מעמודת `טקסט` מוזרק לתבנית קבועה (כפי שקיים היום):
```html
<div class="laura-desc" dir="rtl" style="font-family: inherit; line-height: 1.7; color: #333;">
  <h3 style="font-size: 1.1em; margin-bottom: 10px; color: #222;">{{TITLE}}</h3>
  <p style="margin-bottom: 12px;">{{טקסט}}</p>
</div>
```
- `{{TITLE}}` = כותרת המוצר (בלי מידה). `{{טקסט}}` = תוכן עמודת `טקסט`.
- אם `טקסט` ריק → פסקה ריקה / בלי `<p>`.

## 5. תגיות (`tags`)
- `tags` = `תאור משפחה` (זהה ל-`product_type`). זה הכלל היחיד.
- עמודת `חדש` **ריקה בכל הטבלה** → מתעלמים לגמרי, לא נגזרת ממנה תגית.
- *(כ-5% מהמוצרים נושאים גם תגית שיווקית "חדש" מהעבר — לא קשורה למיפוי; בעדכון פשוט לא לדרוס תגיות קיימות.)*

## 6. מחיר (דרישה) + Reference בלבד
- **מחיר — ✅ דרישה (שינוי מגרסה קודמת):** `variant.price = מחיר מומלץ` כמו שהוא, ללא נוסחה, בזמן היצירה. (הערה היסטורית: נצפתה נוסחה `round(מחיר בסיס × 1.77, 2)` = מרווח 50% + מע"מ 18% ב-92% מהמוצרים — **לא** בשימוש כעת.)
- **מלאי (reference):** נוצר עם **0**; הסריקה השעתית הקיימת מיישבת מלאי אמיתי בטיק הבא. עמודת `מלאי זמין` בקובץ **מתעלמים** ממנה.
- **status (active/draft), מחיקות, תדירות סנכרון** — האוטומציה הקיימת.

## 7. רצף API ליצירת מוצר חדש
```
1. GET  /products.json           (חיפוש SKU קיים → אם קיים: דלג, לא יצירה)
2. GET  /custom_collections.json (מיפוי שם→id; תת-קטגוריה חסרה → POST ליצירה + סימון is_new_collection)
3. POST /products.json           (title, body_html, vendor, product_type, tags, variants[+price], images[])  → status: draft
4. POST /collects.json           → collection_id 477920559358  (קטגוריית על)
5. POST /collects.json           → collection_id של תת-הקטגוריה (מ-שלב 2)
```
```json
{ "product": {
  "status": "draft",
  "title": "{{תיאור פריט ללא מידה}}",
  "body_html": "{{תבנית §4}}",
  "vendor": "לורה סוויסרה | laura swisra",
  "product_type": "{{תאור משפחה}}",
  "tags": "{{תאור משפחה}}",
  "options": ["מידה"],
  "variants": [
    { "option1": "{{מידה}}", "sku": "{{מקט}}", "barcode": "{{ברקוד}}", "price": "{{מחיר מומלץ}}" }
  ],
  "images": [{ "src": "{{קישור לתמונה}}" }]
}}
```

## §9 — תוצאות ולידציה (הרצת הכללים הפוך מול האתר, 812 וריאנטים)

| שדה/כלל | דיוק | הערה |
