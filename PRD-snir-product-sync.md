# מסמך דרישות — ייבוא וסנכרון מוצרי שניר ל-Shopify

> **מטרה:** להגדיר, שדה-אחר-שדה, איך מוצר מאתר הספק **שניר** (snir-bebe.com) מיובא ל-Shopify ומסונכרן.
> **חלק ממערכת Supplier Product Sync.** אומת מול החנות החיה על מדגם רחב (טאבים↔metafields), שדה-שדה.
> **מחיר:** מייבאים את **מחיר הספק כמו שהוא** ל-`variant.price`. המשתמש משנה אח"כ עם הנוסחה שלו (לא חוסם ייבוא).

---

## 0. מקור הנתונים — היברידי מאחורי חומת אנטי-בוט (⚠️ קריטי)
לשניר **WooCommerce Store API ציבורי**, אבל **כל האתר מוגן ב-WAF שמריץ JS-challenge** (דף MD5 שמחשב cookie וטוען מחדש). בקשת `urllib`/`requests` רגילה מקבלת **דף אתגר, לא JSON**.
```
פתרון: דפדפן headless (Playwright/Chrome) — נכנס פעם אחת לאתר → פותר את האתגר → מקבל cookie.
        מאותו רגע: fetch פנימי (same-origin) מחזיר JSON וגם דפי מוצר.
API:  GET /wp-json/wc/store/v1/products?per_page=100&page=N   (דרך הדפדפן)
דף:   GET <permalink>  →  לפרסר .woocommerce-Tabs-panel (הטאבים)
```
**חובה למשוך את שניהם** — ה-API לשדות המובנים, דף ה-HTML לטאב `tech_details` (מאפיינים).
**היקף:** ~330 מוצרים באתר (292 עם מק"ט), 199 בחנות.

---

## 1. מפתח זיהוי
**מק"ט** (`sku` מה-API ↔ `variant.sku`). אומת: **94%** חפיפה (167/178). **פורמט: slug טקסטואלי** (למשל `room-dor+ted-white`, `sh-asenta`, `cl001`) — לא מספרי כמו סגל.
- 11 מק"טים בחנות שאין באתר = מוצר ששניר הוריד → לסמן לבדיקה, **לא למחוק**.
- 1 מוצר-אתר = 1 מוצר-חנות. באתר יש `variable` (200) אבל **190 מהם עם וריאציה אחת בלבד** → בחנות כולם single-variant.

---

## 2. מיפוי השדות (שדות API)
| שדה API | → שדה Shopify | כלל |
|---|---|---|
| `name` | `product.title` | **בסיס** — בחנות נערך ידנית (~62% זהה): מוסיפים "NEW COLLECTION"/צפי-חזרה, לע' משמיטים מידות. לייבוא: name as-is, המשתמש מעשיר |
| `short_description` | `product.body_html` | **as-is (100%)** — זהו ה-body_html בחנות |
| `description` | Metafield `custom.view_productss` (תיאור פריט) | **as-is (94%)** — זהה לטאב `description`. אין צורך בדף לשדה זה |
| `sku` | `variant.sku` | מפתח |
| `prices.regular_price` | **`variant.price`** (as-is) + Metafield `supplier_price` | ⭐ **לייבא כמו שהוא** ל-price; לשמור גם כ-supplier_price (reference). המרה לפי `currency_minor_unit` |
| **`is_in_stock`** | **`variant.inventory` / זמינות** | ⭐ **בוליאני בלבד!** שניר **לא חושף כמות** (`add_to_cart.maximum=9999` תמיד). ראה §5 |
| `images[].src` | `product.images[]` | למשוך ולהעלות (הורדה, לא hotlink) |
| `categories[]` | `product_type` + אוסף | §3 (קטגוריות שיווקיות מסננים) |
| `permalink` | Metafield `supplier_product_url` | עוגן |
| *(קבוע)* | `vendor` = `שניר \| snir` · `status` = **Draft** (חדש) | |
| *(לפי סוג)* | `template_suffix` | §2.1 |
| `barcode` | `variant.barcode` | לבדוק אם קיים ב-API (בד"כ ריק) — לא לצפות לו |

### 2.1 כלל `template_suffix`
| מזהה | → template | דיוק |
|---|---|---|
| שם מתחיל ב-**מיטת/עריסת** | `furniture-beds` | 28/28 (100%) |
| שם מתחיל ב-**שידה/שידת/ארון** | `furniture-product-page` | 55/55 (100%) |
| product_type ∈ {שידות החתלה, ארונות, מזרונים} | `furniture-product-page` | 100% |
| product_type = **מיטות תינוק** | `furniture-beds` | 46/57 (81%) |
| **חדר** (סט חדר) | `furniture-product-page` (ברירת-מחדל) | 50/58 (86%) |
> מקרי "חדר"/"מיטות תינוק" מעורבים (~19) → ברירת-מחדל + **דגל לבדיקה ידנית** ב-Draft.

## 3. קטגוריות
### 3.1 קטגוריית-אתר → `product_type` (לפי מילת-מפתch; מתעלמים משיווקיות)
| קטגוריה/מילה באתר | → product_type |
|---|---|
| מיטות תינוק · מיטת תינוק · מיטות תינוק כולל מעבר | **מיטות תינוק** |
| שידות אחסנה · שידות לתינוק (רוחב…) | **שידות החתלה** |
| חדרי תינוקות · קטלוג חדרים · חדרים-שידות … | **חדרי תינוקות** |
| ארונות | **ארונות לחדרי ילדים** |
| גודל מזרן · מזרן | **מזרונים לתינוק** |
> **מסננים (לא product_type):** `MIX AND MATCH`, `מוצרים בדף הבית`, `קטלוג חדרים` (שיווקיות).

### 3.2 `product_type` → אוסף
מיטות תינוק→מיטות תינוק · שידות החתלה→שידות החתלה · חדרי תינוקות→חדרי תינוקות · ארונות לחדרי ילדים→ארונות לחדרי ילדים · מזרונים לתינוק→מזרונים.
> אין Smart Collections → `collects` מפורש. קטגוריה לא מזוהה → Draft + סימון.

## 4. Metafields — טאבים בעלי-שם (הליבה של שניר)
דף המוצר בנוי מ**טאבים/אקורדיון של WooCommerce** (`.woocommerce-Tabs-panel--<name>`). בשונה מסגל (מיקומי) — כאן המיפוי **לפי שם הטאב**, יציב:

| טאב בדף (`woocommerce-Tabs-panel--`) | תוכן | → metafield | אימות |
|---|---|---|---|
| `tech_details` | פרטים טכניים ("מידה כללית: רוחב… •חומר גלם…") | **`custom.infoo`** (מאפיינים) | **100%** (ריק אם הטאב חסר) |
| `description` | תיאור פריט | **`custom.view_productss`** (= API `description`) | **94%** |
| `oc_theme_product_tab_2` | אחריות | **`custom.securingg`** | **100%** — קבוע (ראה למטה) |
| `oc_theme_product_tab_0` | תנאי משלוחים | ❌ **לא משתמשים** — delivery בחנות הוא boilerplate קבוע | |
| `oc_theme_product_tab_1` | תנאי תשלום | ❌ **נזרק** | |

**המרה ל-rich_text:** כל טאב → `rich_text_field` (JSON), **כל שורה (`<br>`/`<p>`) → `paragraph` נפרד**, as-is (בלי עריכה). זהה לכלל של סגל.

### 4.1 ערכים קבועים (constants — לא מהאתר)
| metafield | ערך |
|---|---|
| `custom.securingg` (אחריות) | **קבוע** (199/199): `כל המוצרים מגיעים עם 12 חודשי אחריות לפי התקנון.` |
| `custom.delivery` (משלוחים) | **boilerplate קבוע** (185/199): "שירות הובלה והרכבה לריהוט וחדרי תינוקות…". **חריג:** מוצרי קולקציית **"סטודיו בוטיק"** — מוסיפים מחירון ("שידה-400 ש"ח, מיטה-350, שידה+מיטה-600") |
| `global.title_tag` | ← name |
| `global.description_tag` | ← description |

> ⚠️ **אחריות שונה מסגל:** שניר = **12 חודשים** (סגל = 5 שנים). קבוע לכל המוצרים.
> *(סנכרון: `supplier.*` — supplier_sku/price/url/stock/timestamps — פנימי.)*

---

## 5. מלאי — בוליאני בלבד (⚠️ שונה מסגל)
שניר **לא חושף כמות מלאי** (`add_to_cart.maximum=9999` לכולם; `low_stock_remaining` רק ל-4 מוצרים). המקור היחיד האמין = `is_in_stock`.
| מקור | → פעולה |
|---|---|
| `is_in_stock = true` (276) | זמין למכירה |
| `is_in_stock = false` (54) | Out of Stock (לא למחוק) |
> אין מספר לסנכרן — מנהלים זמינות בוליאנית. (בסגל היה מספר → שם סנכרנו כמות; כאן אין.)

---

## 6. מחיר
מייבאים את `prices.regular_price` **כמו שהוא** אל `variant.price`, ושומרים עותק כ-`supplier_price` (reference). **האוטומציה לא מחשבת נוסחה** — המשתמש משנה את מחיר-המכירה אח"כ ידנית עם הנוסחה שלו. `compare_at_price` אופציונלי (המשתמש). *(תצפית בלבד, לא בשימוש: מחיר-חנות היה ≈ ספק×0.90.)*

## 7. שני תהליכים
- **ייבוא:** מוצר-אתר בלי SKU תואם → Draft + כל §2-§5. סקופ: ~292 באתר מול 178 בחנות.
- **סנכרון (מק"ט):** זמינות (`is_in_stock`) · מחיר-ספק (היסטוריה+התראה) · לוג לכל שינוי. תדירות 3-6ש'.

## 8. מקרי קצה
| מצב | טיפול |
|---|---|
| דף מחזיר challenge (WAF) | לרענן cookie דרך הדפדפן; retry על תגובה שאינה JSON |
| מק"ט קיים | עדכון (זמינות/מחיר), לא יצירה |
| 11 מק"טים בחנות שאין באתר | שניר הוריד — לסמן, לא למחוק |
| טאב `tech_details` חסר | `custom.infoo` ריק (תקין — ~חלק מהמוצרים) |
| template מעורב (חדר/מיטות תינוק) | ברירת-מחדל §2.1 + דגל לבדיקה |
| קטגוריה שיווקית בלבד (MIX AND MATCH…) | להתעלם, לגזור product_type ממילות-מפתח אחרות |
| מוצר "סטודיו בוטיק" | delivery = boilerplate + מחירון הקולקציה |

## 9. מה על המתכנת להשלים (Implementation TODO)
מעבר למיפוי (מכוסה למעלה) — נדרש במימוש:
1. **זיהוי מה חדש (dedup):** להשוות מק"ט(slug)-ספק↔חנות (94%) → לייבא רק חדשים; קיים → עדכון.
2. **⚠️ דפדפן חובה (WAF):** את המשיכה (API + דפים) לבצע דרך דפדפן headless (Playwright) שפותר את ה-JS-challenge — §0. קריאת HTTP רגילה תיכשל ותחזיר דף אתגר.
3. **וריאציות:** ~10 מוצרים עם 2-3 variations — כלל לפיצול/בחירה (בחנות single-variant).
4. **תמונות:** הורדה+העלאה ל-Shopify CDN.
5. **template מעורב:** מקרי "חדר"/"מיטות תינוק" → ברירת-מחדל §2.1 + דגל לבדיקה. לאמת שקבוע האחריות (12 חודש) תקף למוצר חדש.
